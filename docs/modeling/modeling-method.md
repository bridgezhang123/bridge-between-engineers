# Modeling Method / 建模方式 - Top-down 与 Bottom-up

> 在很多项目里，建模效率高低并不取决于画图速度，而取决于一开始是否选对了建模方式。

## 1. Scope and Goal / 范围与目标

本文讨论 `Top-down` 与 `Bottom-up` 两种常见建模方式，目标不是简单分出高下，而是帮助判断：

- 哪些场景适合联动建模
- 哪些场景适合独立建模
- 如何在个人或小团队环境下保持层级清晰

## 3. Practice and Templates / 实操与模板

### 3.1 Bottom-up / 自底向上

适合以下对象：

- 标准件
- 外购件
- 尺寸基本固定的成熟零件

优点是结构独立、依赖少、后期迁移容易。缺点是联动修改成本较高。

### 3.2 Top-down / 自上而下

适合以下对象：

- 配合尺寸强关联的零件
- 需要一起调整孔位、定位面和界面尺寸的结构
- 布局已较明确的核心装配

优点是改一处可带动多处更新。缺点是依赖关系更复杂，层级不清时容易出错。

### 3.3 A Practical Rule / 一个更稳妥的经验

对个人或小团队而言，更稳妥的方式通常不是“全 Top-down”或“全 Bottom-up”，而是：

- 标准件、外购件尽量 Bottom-up
- 核心配合件适度 Top-down
- 子装配体内部保持清晰的依附关系
- 总装层级尽量避免无节制地下放关联

## 4. Other Key Points / 其余要点

### 4.1 Level Awareness / 层级意识

在装配体中编辑时，应始终留意当前处于哪个层级，是总装、子装配体，还是单个零件。层级判断错误时，外部参考与位置关系往往最容易混乱。

### 4.2 Through Features / 贯穿特征

像贯穿孔这类特征，首先应判断它属于哪个层级，再决定是使用装配体特征还是零件内部关联。先判断层级，再建模，通常更稳妥。

## 5. Boundaries and Risks / 边界与风险

- 子装配体中的零件若直接依赖总装层级，后续调整时容易发生引用混乱
- 同一组尺寸若跨越多个层级创建关联，排错成本会显著上升
- Top-down 用得过多时，模型的可迁移性和可维护性会变差

## 6. Summary / 小结

建模方式没有绝对优劣，关键在于是否与对象类型、层级关系和修改频率相匹配。对个人或小团队来说，通常是适度混用，而不是坚持单一方法。
