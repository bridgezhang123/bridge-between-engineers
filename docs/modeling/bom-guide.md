# BOM 表

> BOM 表的价值，不只在于列出零件名称，更在于让设计、采购、装配与后续维护看到同一份结构信息。

## 1. 范围与目标

本文希望说明：

- BOM 与模型、图纸之间的关系
- 哪些字段值得优先统一
- 如何降低手工维护带来的错误

## 2. 标准引用

暂无。

## 3. 实操与模板

### 3.1 建议优先统一的字段

- Part Number
- Description
- Quantity
- Material
- Remark

### 3.2 实操原则

- 先统一命名，再谈 BOM 稳定输出
- 装配层级应尽量清楚，避免一张表里混杂多套逻辑
- 若某些字段后续要供采购或 ERP 使用，更应提前规范

### 3.3 常见输出

- 模型自动生成的 BOM
- 配合 Excel 管理的 BOM
- 用于采购、装配或检验的派生清单

## 4. 其余要点

### 4.1 命名优先

如果命名本身就不稳定，BOM 再完整也很难真正长期可用。

### 4.2 跨部门沟通

BOM 不只是给设计人员自己看的，它往往也是采购、装配与维护理解产品结构的第一入口。

## 5. 边界与风险

- 名称不规范时，BOM 再完整也难以长期维护
- 手工补表过多时，模型与表格容易脱节
- 爆炸视图序号与 BOM 若不同步，阅读体验会明显下降

## 6. 小结

BOM 的关键不在于表格本身，而在于它是否真实反映了装配结构，是否能被设计、采购和装配共同使用。结构越清楚，BOM 的价值越大。
