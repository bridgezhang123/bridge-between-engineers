# Modeling & Drawings / 建模与图纸

本栏目面向个人工作者，或尚未建立成熟 PDM 体系的小团队，尝试从建模、出图与版本管理的角度，逐步引发对标准化建模的思考与实践。

!!! abstract "Series Goal / 系列目标"
    这组文章并不假定读者已经拥有完整的信息化管理条件，而是从更现实的起点出发：先把命名、目录、建模方法、图纸表达和版本迭代这些基础环节梳理清楚。

## Suggested Reading Order / 建议阅读顺序

1. [Naming Standards / 命名标准](naming-standards.md) - 从零部件标准化的命名开始（包含文件夹命名）
2. [Modeling Method / 建模方式](modeling-method.md) - 建模方式对比：Top-down(自上而下) vs Bottom-up(自底向上)
3. [Parametric Modeling / 参数化建模](parametric-modeling.md) - 浅析参数化建模
4. [Configurations / 配置功能](configurations.md) - 配置功能的使用及其边界
5. [Drawing Template / 图纸模板](drawing-template.md) - 图纸规范--模板解析
6. [Dimensioning Basics / 基本标注](dimensioning.md) - 图纸规范--标注的基本要求
7. [Exploded Views / 爆炸视图](exploded-view.md) - 图纸规范--爆炸视图的使用
8. [BOM Practice / BOM 表](bom-guide.md) - 图纸规范--BOM表的使用
9. [Revision Control / 版本迭代](revision-control.md) - 以文件的版本迭代管理结束

## Writing Principle / 写作原则

这组文章尽量采用相近的推进方式：

- 先说明问题为何存在
- 再交代适用范围与目标
- 然后给出可直接落地的实操建议
- 最后补充边界、风险与参考来源

这样做的目的，是让每篇文章都既能单独阅读，也能放回整个系列中理解。
