# 爆炸视图

> 爆炸视图不是为了“看起来更复杂”，而是为了更清楚地表达装配关系、顺序与层级。

## 1. 范围与目标

本文讨论：

- 爆炸视图适合用在什么场景
- 配置、步骤与二维图纸如何配合
- 如何避免为了“炫”而增加无效工作

## 2. 标准引用

暂无。

## 3. 实操与模板

### 3.1 适用场景

- 装配说明
- 维修说明
- 带序号的总装图

### 3.2 实操原则

- 先建立专门的爆炸视图配置
- 爆炸步骤尽量按真实装配逻辑组织
- 与序号标注和 BOM 保持一致

### 3.3 常见流程

- 先建立配置
- 再创建 exploded steps
- 最后在二维图纸中调用相应视图

## 4. 其余要点

### 4.1 步骤顺序很重要

爆炸视图若不按装配逻辑组织，即使看起来“炸开了”，阅读价值也不高。

### 4.2 与 BOM 的关系

若一张总装图既带爆炸视图，又带序号和 BOM，那么三者的顺序与名称最好保持一致。

## 5. 边界与风险

- 爆炸方向若缺乏层次，会降低可读性
- 为爆炸而爆炸，会增加维护成本
- 若配置管理不清，爆炸视图容易与正常装配状态混淆

## 6. 小结

好的爆炸视图不在于动作多，而在于结构清楚、顺序合理、能和序号及 BOM 自然配合。能帮助阅读的爆炸视图，才值得保留。
